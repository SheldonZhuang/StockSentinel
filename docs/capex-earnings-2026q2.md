## 三、系统自动行为（无需人工干预的部分）

1. **数据接入全自动**：每日 cron 拉 EDGAR XBRL。各家 10-Q/10-K 正式申报后（通常滞后
   新闻稿发布数日），单季数据自动进入 `calcCapexQuarterYoY`。
2. **错季对齐保护**：四家**全部**申报完成前，系统共同季保持 2026Q1 不动（防混算）——
   预计 8 月上旬（MSFT 10-K 最慢）才会整体翻新到 2026Q2。财报季中途前端显示旧季属正常。
3. **判定自动生效**：翻新当日 N1/N2 侦察兵规则自动用新数据判定；若单季转负→capex 子信号
   封顶 neutral（进攻档立即不可达）；两季连负→直接收紧。当前基数下 Q2 转负需要合计
   < $88.2B（较 Q1 环比 -32%），概率极低——本季的真实看点是**增速二阶变化**（80.5% 之后是加速还是降速）。
4. **展示自动更新**：前端维度卡括号、参考指标行、AI 产业链面板、日报事实清单同步翻新。
5. **指引检测 + 财报快报全自动（2026-07-23 起，113号）**：新业绩 8-K 检出当日，新闻稿与
   web 检索（电话会实录/PPT/主流财经媒体）双源并行判定指引方向，并生成单公司快报
   （单季/TTM 额度与同比——新季值 FMP 优先、新闻稿 LLM 提取兜底，不等 10-Q）入档展示。
   明确下修+高置信自动录 N3（新闻稿源即刻录；web 源须佐证：管理层原话或 ≥2 独立来源）。
   注意快报为展示口径；N1/N2/TTM 判定层仍以 EDGAR XBRL 为准（可审计原则）。
