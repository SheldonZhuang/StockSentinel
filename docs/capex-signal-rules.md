# 云厂商 Capex 判定规则总览（AI 供需维·投资侧）

> 建档 2026-07-21。汇总 capex 子信号的全部判定规则（N1/N2/N3）、数据源冗余、展示位置
> 与关键设计决策。capex 是 AI 供需现金流三件套的第二环（模型调用量→**云capex**→半导体产出）。

## 一、数据口径与来源

| 口径 | 含义 | 判定角色 | 数据源 |
|---|---|---|---|
| **TTM 同比** | 滚动4季资本开支同比 | **主判定口径** | SEC EDGAR XBRL（主）+ FMP 现金流量表（备源，EDGAR 单家失败自动补位） |
| **最新单季同比** | 四家共同已披露最近日历季合计同比（错季对齐） | 拐点侦察兵（N1/N2） | 同上 |
| **capex 指引** | 财报新闻稿中的前瞻性 capex 指引方向 | 前瞻事件（N3） | EDGAR 8-K 业绩发布 EX-99 新闻稿 → LLM 识别（自动）+ 人工兜底 |

阈值（`signal.config.js`）：TTM/单季同比 >+10% 宽松 / <0% 收紧 / 其间中性。

## 二、判定规则 N1/N2/N3

**基线**：capex 子信号 = TTM 同比按阈值分档。以下规则按优先级从低到高叠加：

- **N1 单季拦截宽松**（2026-07-20）：最新单季同比 <0 时，capex 子信号不得判宽松（最多中性）。
  同构先例=资产负债表 QT 只拦截宽松不单独判收紧。null 处理对称（TTM 缺失+单季负→中性）。
- **N2 两季连负判收紧**（2026-07-20）：连续两个财报季单季同比 <0 → capex 直接判收紧，
  不等 TTM 转负（比 TTM 提前 1-2 个财报季）。两季连负基本排除单季交付节奏噪声。
- **N3 指引下修事件**（2026-07-21，优先级最高）：巨头财报明确下修 capex 指引 → capex 子信号
  强制收紧，覆盖 N1/N2 与一切数据口径。这是"实际数据还在增长但指引已转向"的前瞻捕获。
  自动检测（LLM 判新闻稿）"明确下修+高置信"自动录入；电话会口头下修人工录入兜底。

capex 子信号收紧 → 三件套共识规则"任一环节收紧→整维收紧" → AI 供需维收紧。

## 三、关键设计决策：指引下修 → 减仓观望，不直达防守（2026-07-21 用户确认）

用户提议："指引下修=云capex级别收紧=AI供需供过于求=**直接进入防守**"，推理链正确，
但最后一步（单维直达防守）经评估**维持现状为减仓观望**，三条理由：

1. **回测实证否决单维直达防守**：防守分级（2026-07-12）前单维收紧即全面防守→防守占比74%、
   假阳性泛滥；分级后实证单维收紧月份市场月均收益（+1.2%）不低于其他月份（+1.5%）——
   单维收紧是关注信号非行动信号。给指引下修开单维直达防守特例=在分级体系上凿洞。
2. **信号源可靠性配不上防守代价**：指引判定来自 LLM 解析新闻稿文本（全系统唯一非纯数值输入）。
   误判在防守档=卖出全部存量TQQQ+30天迟滞锁定；在减仓档=仅停止加仓。轻决策交文本、
   重决策交数据确认，是风险配置纪律。
3. **真实下修几乎不单独出现**：巨头真下修时市场当天反应，模型调用量/半导体链/半导体产出
   会跟进→第二环收紧（双维共振）自然触发全面防守，通常只差数天到数周。前瞻信号做预警
   排头兵，数据确认扣扳机——与"单季侦察兵 vs TTM 确认官"同一哲学。

**当前效果链**：指引下修 → 当天邮件+红横幅通知 → AI供需维收紧 → 至少减仓观望（停止加仓）
→ 市场数据跟进确认 → 双维收紧 → 全面防守（S5 卖出指令）。

> 复议前提：等实盘正确触发过一次、验证 LLM 判定质量后再议是否升级为直达防守。
