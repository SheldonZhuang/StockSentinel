# 股哨兵 Stock Sentinel 设计文档

日期：2026-07-05

## 1. 产品定位

**股哨兵 Stock Sentinel** 是一个美股投资进攻/防守信号判断与示警工具。

核心理念：
- 长线遵循奥地利经济学派（供需决定价格，主观价值论）——工具本身不判断"买什么"，交由用户根据"优质国家/龙头产业/龙头企业"的选股原则自行决定标的。
- 短线遵循政治经济学（政策扰乱市场价格，决定进攻/防守时机）——工具的核心价值在于把货币政策、财政政策、行政政策的松紧信号转化为清晰的进攻/中性/防守三档信号，帮助用户判断"什么时候买、什么时候卖"。
- 进攻 = 买在无人问津处 = 政策松无可松时；防守 = 卖在人声鼎沸时 = 政策开始收紧时。不论进攻或防守，一定要果断。

**范围边界**：仅覆盖美股（不含房产，房产由已有的 HousingSentinel/GlobalHousingSentinel 系列独立处理）。

**判断粒度**：市场整体信号（进攻/中性/防守）+ 个股/ETF 级别的价格位置与估值评估（辅助判断具体标的贵贱，不做买卖建议）。

## 2. 信号计算逻辑

信号计算分两层：**信号位判定**（货币/财政/行政三个独立信号位，各自判定为宽松/中性/收紧）→ **决策树合成**（三个信号位组合成最终进攻/中性/防守）。

### 2.1 货币信号位（自动计算）

数据源：[FRED](https://fred.stlouisfed.org/)（美联储圣路易斯分行经济数据库，免费公开 API，无需 key 或低门槛注册即可获取）。

拉取指标：
1. 联邦基金利率目标区间（决定加息/降息/暂停及单次调整幅度）
2. 联储资产负债表总规模（[WALCL](https://fred.stlouisfed.org/series/WALCL) 系列，决定 QE 扩张/QT 收缩/暂停）
3. 核心 PCE 同比（[PCEPILFE](https://fred.stlouisfed.org/series/PCEPILFE)，仅展示参考，不进入判定）
4. Trimmed Mean PCE（[PCETRIM1M158SFRBDAL](https://fred.stlouisfed.org/series/PCETRIM1M158SFRBDAL) 等达拉斯联储系列，仅展示参考，不进入判定）
5. 失业率（[UNRATE](https://fred.stlouisfed.org/series/UNRATE)，仅展示参考，不进入判定）

**货币信号位决策规则**：
- **宽松**：利率方向为暂停或降息（非加息）**且** 资产负债表不收缩（QE 扩张或规模暂停，非 QT）
- **收紧**：利率为应对式加息（单次调整幅度 ≥ 50bp）**或** 资产负债表处于 QT 收缩状态
- **中性**：其余情况（例如利率为预防式加息 <50bp 但资产负债表未扩张等混合情况）

指标 3/4/5（核心 PCE、Trimmed Mean PCE、失业率）**不参与判定逻辑**，仅在前端以数值+趋势展示，供用户参考验证货币信号位判断是否合理。

### 2.2 财政信号位（人工设定）

覆盖范围：减税降费/加税加费、大基建投资增减、特别国债增减、消费券发行增减、财政赤字上限调整等。

参考素材（自动抓取，仅供管理员参考，不自动定档）：
- [Federal Register API](https://www.federalregister.gov/developers/documentation/api/v1)：抓取涉及财政的行政令/公告/规则
- 美国财政部新闻稿页面：https://home.treasury.gov/news/press-releases/index （无公开 API，抓取标题列表）

**设定方式**：管理员在后台管理界面直接设定财政信号位当前档位（宽松/中性/收紧），并指定有效期。到期后自动回归中性。

### 2.3 行政信号位（人工设定）

覆盖范围：金融/科技监管松紧、罚款增减，以及贸易战/关税战/科技战的开始、恶化、缓和、结束（作为行政信号位的子类别，不单独拆出信号位）。

参考素材（自动抓取，仅供管理员参考）：
- Federal Register API（同上，抓取涉及监管/贸易/科技限制的行政令）
- 白宫新闻/行政令页面：https://www.whitehouse.gov/news/ 、https://www.whitehouse.gov/presidential-actions/executive-orders/ （无公开 API，抓取标题列表）
- 商务部新闻页面：https://www.commerce.gov/news/press-releases （无公开 API，抓取标题列表）

**设定方式**：与财政信号位相同——管理员后台直接设定当前档位 + 有效期，到期回归中性。

> 注：财长/商务部长等官员的社交媒体账号不作为自动化数据源接入（存在账号被封、被仿冒等真实性风险），仅作为管理员人工判断时的参考信息来源，由管理员自行核实。

### 2.4 最终信号合成（决策树，非对称）

- **进攻**：货币、财政、行政三个信号位**全部**为宽松
- **防守**：三个信号位中**任意一个**为收紧
- **中性**：其他情况（无收紧，但非全部宽松）

设计意图：防守信号采用 OR 逻辑（宁可错杀不可放过，任何收紧信号出现就立刻防守），进攻信号采用 AND 逻辑（三方面政策必须同时宽松才算真正的"无人问津"买点）。体现"防守要果断"的风格。

### 2.5 个股/ETF 评估维度

数据源：Yahoo Finance（免费，历史价格 + P/E/P/S 等基本面数据）。

每个自选标的展示：
- **价格位置**：当前价格在用户自定义时间范围内的百分位（例如"近3年区间的78%位置"）。数据源 Yahoo Finance 历史价格,可靠地计算真实百分位。
- **估值现状**：当前 P/E、P/S 的实时数值（Yahoo Finance 提供的快照数据）。**不做历史百分位排名**——Yahoo Finance 免费接口只提供当前估值快照,不提供历史 P/E/P/S 时间序列；若用当前 EPS 反推历史 P/E，其百分位排名在数学上会与价格百分位完全一致（同除一个常数不改变排序），是没有额外信息量的伪指标，因此不实现。

时间范围为用户自由选定的起止日期（非固定档位），价格位置指标使用该时间范围计算。

工具不对个股做买卖建议，仅展示位置数据，供用户结合自己的选股判断（优质国家/龙头产业/龙头企业）参考。

## 3. 用户与数据模型

- **登录**：邮箱+密码 注册登录（bcrypt 加密密码），JWT 会话。Google OAuth 一键登录设计为可插拔的 Passport.js 策略，本次 MVP 暂不接入（需 Google Cloud Console 客户端凭据），代码结构预留但不实现，后续补充凭据后再启用
- **自选股清单**：每用户可自定义添加不限数量的股票/ETF；同时提供内置预设清单（Mag7：AAPL/MSFT/GOOGL/AMZN/NVDA/META/TSLA + QQQ + SPY）供快速添加
- **付费订阅**：预留位（数据模型和 UI 预留付费标记字段），MVP 阶段不开启付费墙，所有登录用户可用全部功能
- **信号位/事件数据全局共享**：财政/行政信号位由管理员统一设定，所有用户看到同一份市场信号，不因用户而异
- **管理员权限**：通过环境变量指定的管理员邮箱识别，登录后自动获得后台管理入口，无需额外密码

## 4. 示警与通知

- 当最终信号档位发生变化（例如中性→进攻，或进攻→防守）时，通过 [Resend](https://resend.com/) 发送邮件提醒给已订阅提醒的用户
- 用户注册时默认开启邮件提醒，可在设置中关闭
- 页面上同时用信号灯视觉呈现当前状态（见第 6 节）

## 5. 前端信息架构

**布局**：双栏工作台
- 左栏：固定显示宏观信号面板——顶部信号徽章（进攻/中性/防守 + 综合状态说明），下方货币/财政/行政三个信号位明细，以及货币位内 5 个 FRED 指标的参考数值
- 右栏：固定显示自选股/ETF 清单，含价格位置和估值百分位

**首页**：额外展示信号历史变化时间轴（标注过去信号档位切换的时间点），数据库按日累积计算结果供回溯。

**信号灯视觉风格**：文字徽章——圆角矩形 + emoji + 多语言文字标签（🟢进攻 / 🟡中性 / 🔴防守），当前档位高亮，其余两档淡化显示。

**多语言**：前端静态语言包，覆盖中文、英文、法语、德语、西班牙语、日语、韩语。默认语言根据浏览器 `navigator.language` 自动检测，用户可手动切换并记住选择。多语言仅覆盖 UI 文案（按钮、标签、说明文字），不含动态生成的解释性长文本。

**响应式**：单一 Vue3 + Vite Web 应用，响应式布局适配桌面/手机/平板浏览器，不做独立移动端 App 或小程序。

## 6. 技术架构

沿用 HousingSentinel/GlobalHousingSentinel 的成熟架构模式。

```
StockSentinel/
├── backend/
│   ├── server.js                    # Express 入口，路由 + cron 任务注册
│   ├── config/
│   │   └── signal.config.js         # 决策树规则、指标阈值等可调参数
│   ├── api/
│   │   ├── fetch-macro.js           # FRED API 拉取（利率、资产负债表、PCE、失业率）
│   │   ├── fetch-stocks.js          # Yahoo Finance 拉取（价格、P/E、P/S）
│   │   ├── fetch-federal-register.js # Federal Register API 拉取（供后台参考）
│   │   ├── signal.js                # 信号位判定 + 决策树合成逻辑
│   │   ├── admin.js                 # 后台管理接口（设定财政/行政信号位 + 有效期）
│   │   ├── auth.js                  # JWT 登录、Google OAuth
│   │   └── watchlist.js             # 自选股增删查
│   └── utils/
│       ├── storage.js               # SQLite 读写封装
│       └── mailer.js                # Resend 邮件发送
├── frontend/
│   └── src/
│       ├── App.vue
│       ├── i18n/                    # 静态语言包 zh/en/fr/de/es/ja/ko
│       └── components/
│           ├── SignalBadge.vue       # 信号灯文字徽章
│           ├── MacroPanel.vue        # 左栏宏观信号面板
│           ├── WatchlistPanel.vue    # 右栏自选股面板
│           └── SignalTimeline.vue    # 首页信号历史时间轴
├── nixpacks.toml                    # Railway 部署配置
├── vercel.json                      # 前端 SPA 路由配置
└── CLAUDE.md
```

| 层 | 选择 | 原因 |
|---|---|---|
| 数据库 | SQLite | 单文件，Railway Volume 持久化，零成本 |
| 后端 | Express 单进程 | 抓取+API+cron 一体，与已有项目一致 |
| 前端 | Vue3 + Vite → Vercel | 零配置 CI/CD |
| 认证 | JWT + Google OAuth | 无状态，跨设备通用 |
| 邮件 | Resend | 免费额度每日3000封，Node生态集成简单 |
| 部署 | Railway（后端）+ Vercel（前端） | 与已有项目一致，经验可复用 |

**定时任务（cron，UTC 时区）**：
1. 每日拉取 FRED 宏观数据、Yahoo Finance 股票数据、Federal Register 参考素材
2. 计算货币信号位 → 结合财政/行政信号位当前设定 → 决策树合成最终信号
3. 若财政/行政信号位任一到期，自动回归中性后重新合成
4. 若最终信号档位与前一日不同，触发 Resend 邮件通知所有开启提醒的用户
5. 计算结果存入 SQLite，供前端读取与历史时间轴回溯

**容错处理**：若某日 FRED 或 Yahoo Finance API 拉取失败，前端沿用上一次成功拉取的数据和信号，并在页面标注"数据更新于 XX 日期"，不中断服务展示。

## 7. 后台管理界面

- 访问权限：登录用户的邮箱与环境变量 `ADMIN_EMAIL` 匹配即可访问，无需额外密码
- 功能：
  - 查看 Federal Register / 财政部 / 白宫 / 商务部 抓取的最新公告标题列表（供参考判断）
  - 设定财政信号位当前档位（宽松/中性/收紧）+ 有效期
  - 设定行政信号位当前档位（宽松/中性/收紧）+ 有效期
  - 查看信号位设定历史记录

## 8. MVP 范围之外（后续迭代）

- 移动端原生 App 或小程序
- 前后端同时支持多语言的动态内容（当前仅 UI 静态文案）
