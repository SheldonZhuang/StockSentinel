## 一、投资人视角的核心结论

**这套系统的真实身份是"防守哨兵 + 纪律执行器"，不是预测器。** 26.4年回测的价值曲线几乎全部来自
五次大危机的规避（召回5/6）；精确率（假阳性71%）是为召回付出的保费，已通过防守分级、趋势再入场、
降档迟滞把保费压到结构最优附近。以下按"若我管自己的钱"的标准列出剩余的真实风险，按期望损失排序：

### 1. 运营风险 > 信号风险（本轮最大发现，已修复）
一个防守信号系统最贵的故障不是误报，是**危机那一周恰好静默停摆**——沉默与"无变化"不可区分。
审查发现三处此类缺口，全部修复：
- cron 失败只写日志无告警 → 现在 FRED 失败/未捕获异常均发管理员运维告警邮件；
- 前端对过期快照无警示 → 现在快照 ≥3 天未更新显示红色失效横幅（7语言）；
- S5 执行邮件（交易命门指令）排在产业链快照/日报/备份之后，中途崩溃则指令永久丢失 →
  已前移至快照落库后立即发送，且发送三连败升级为运维告警双通道。

### 2. 数据资产风险（已修复）
- Railway 文件系统非持久化，备份只上传不恢复——重部署即丢 track record（产品信誉资产）→
  已加启动时从 GitHub 备份自动恢复；
- 用户注册/自选/override/API key 只随每日 cron 备份，丢失窗口最长24小时 → 写后防抖60秒即时备份；
- 滚动部署新旧容器双写 latest.db 竞态 → 上传带快照日期守卫。

### 3. 判定链"漏防守"侧缺陷（已修复，均为数据故障日的边界路径）
- **萨姆锁误解锁**：SAHM 数据缺失日恰逢 <50bp 利率调整会误解锁（null 被当作"已回落"）→
  fail-closed：缺数日已激活的锁视同触发存续（线上与回测复刻同步修改，回测成绩逐位不变）；
- **N1 对 null 不对称**：TTM 缺失而单季转负时 capex 子信号为 null（被共识剔除，usage+semi
  双宽松即可开进攻）→ 改为 neutral，与"单季收缩最多观望"语义一致；
- **stale-keep 吞掉 N2**：AI 维数据缺失判定不含单季通道，两季连负的收紧票可能被旧信号覆盖 → 已纳入。

### 4. 数据管道潜伏缺陷（已修复）
- capex TTM 位置切片在季度序列中段缺口时会拼错季，产出**看似合理的错误同比而非 null**
  （实证构造复现：缺 2024Q3/Q4 的序列输出 0% 假同比）→ 加 8 季连续性校验；
- EDGAR 同期重复 fact 依赖数组顺序取"修正值"，而 API 不承诺顺序 → 改为显式按 filed 日期取最新；
- 次级：market-data 缓存无界增长（写时清扫）、createUser 非原子（事务化）、persist 临时文件名
  跨进程互踩（带 pid）。

### 5. 未修复的已知观察（低危或属设计张力，明示留档）
| # | 观察 | 评估 |
|---|---|---|
| a | FOMC 决议日货币维单日 tight→loose 闪变，次日翻回时会重发一封 dimTight 邮件（连续加息周期每次决议一封） | 最终档不受影响（迟滞吸收）；修复需引入"决议生效日"概念，收益是省一封邮件，暂不动 |
| b | 降档迟滞确认的是"离开旧档满30天"，不是"目标档持续30天"——第30天生效的可能是仅出现1天的候选档 | 线上与回测同口径、校准自洽；语义留档，改语义=重新回测 |
| c | 手动覆盖存续期间，实时 API 直出无迟滞的重算值，与快照口径分裂（含非覆盖维度的降档） | "管理员操作即时生效"的设计代价；覆盖应短期使用，长挂属使用问题 |
| d | cron 停摆 >100 天后端点差退化可产生假应对式锁（RATE_LOOKBACK_DAYS=100 保护半径） | 需连续停摆超100天，运维告警上线后不可能无感知走到这一步 |
| e | 月度回测引擎对 calcMonetarySignal 等有内联复刻（已在文件头注释声明），仅日度回放共享实现 | 漂移风险靠"改判定必重跑回测"纪律覆盖；守卫测试待补 |

### 6. 方法论层的两点诚实披露（已写入 S5 手册）
- **AI 供需维零回测背书**：26年回测中 AI 维全程 neutral（无历史数据），回测的全部往返由政策维/锁
  触发。AI 维触发的防守与进攻档是"规则推演"而非"实证规则"——这恰是唯一的进攻点火引擎，
  使用进攻档时应知道它没有历史成绩单。
- **样本内调参**：V/W/X 三轮变体全部在同一个 n=6 危机样本上选择，存在过拟合风险；n=6 无法做
  留出验证。缓解手段就是公开 track record——从规则冻结日起的每一天都是样本外考试。

### 7. "如果重做一遍我会怎么做"（供长期路线参考，非本轮行动项）
- 保持现有防守体系不动——它已过六轮审查+多轮回测校准，边际改进空间小于引入回归的风险；
- 补的不是新指标而是**双源冗余**：OpenRouter（调用量唯一源）与 EDGAR（capex 唯一源）任一长期
  故障会让 AI 维长期 stale——第二数据源的价值高于任何第五维度；
- 2022 型"预期驱动的顶"（滞后148天）是框架内已知不可达——货币维量的是行动不是预期；接受它，
  用 CAPE 执行层（已启用）而非信号层弥补；
- 执行纪律是收益的一半：S5 手册的"机械执行"承诺没有技术强制力，真正的敌人是连续4次假信号后
  的人。手册已新增两条人工核对规则对冲最贵的两个执行错误。
